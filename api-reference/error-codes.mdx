## Error Response Format

When an API request fails, Creem returns a JSON error response with a consistent structure:

```json
{
  "error": {
    "code": "invalid_parameter",
    "message": "The 'amount' field must be a positive integer.",
    "field": "amount",
    "doc_url": "https://docs.creem.io/api-reference/error-codes#invalid_parameter"
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `code` | string | Machine-readable error identifier |
| `message` | string | Human-readable error description |
| `field` | string | (Optional) The specific field that caused the error |
| `doc_url` | string | (Optional) Link to relevant documentation |

## HTTP Status Codes

| Status | Meaning | When It Occurs |
|--------|---------|----------------|
| `400` | Bad Request | Invalid request parameters or malformed JSON |
| `401` | Unauthorized | Missing API key |
| `403` | Forbidden | Invalid API key or insufficient permissions |
| `404` | Not Found | Requested resource doesn't exist |
| `409` | Conflict | Resource already exists or state conflict |
| `422` | Unprocessable Entity | Valid JSON but semantically incorrect |
| `429` | Too Many Requests | Rate limit exceeded |
| `500` | Internal Server Error | Something went wrong on our end |

## Error Codes Reference

### Authentication Errors

<AccordionGroup>
  <Accordion title="invalid_api_key" icon="key">
    **HTTP Status:** 401 / 403
    
    **Description:** The API key provided is invalid or has been revoked.
    
    **How to fix:**
    - Verify your API key in the [dashboard](https://creem.io/dashboard/developers)
    - Ensure you're using the correct key for the environment (test vs. production)
    - Check that the key hasn't been revoked or regenerated
    
    ```json
    {
      "error": {
        "code": "invalid_api_key",
        "message": "The API key provided is invalid or has been revoked."
      }
    }
    ```
  </Accordion>

  <Accordion title="missing_api_key" icon="lock">
    **HTTP Status:** 401
    
    **Description:** No API key was provided in the request headers.
    
    **How to fix:**
    - Add the `x-api-key` header to your request
    - Ensure the header name is exactly `x-api-key` (lowercase)
    
    ```bash
    curl -X GET https://api.creem.io/v1/products \
      -H "x-api-key: creem_YOUR_API_KEY"
    ```
  </Accordion>
</AccordionGroup>

### Validation Errors

<AccordionGroup>
  <Accordion title="missing_required_field" icon="asterisk">
    **HTTP Status:** 400
    
    **Description:** A required field was not provided in the request.
    
    **How to fix:**
    - Check the API documentation for required fields
    - Ensure all required fields are included in your request body
    
    ```json
    {
      "error": {
        "code": "missing_required_field",
        "message": "The 'product_id' field is required.",
        "field": "product_id"
      }
    }
    ```
  </Accordion>

  <Accordion title="invalid_parameter" icon="circle-exclamation">
    **HTTP Status:** 400 / 422
    
    **Description:** A parameter value is invalid or in the wrong format.
    
    **How to fix:**
    - Verify the data type matches the expected type
    - Check for valid enum values
    - Ensure URLs are properly formatted
    
    ```json
    {
      "error": {
        "code": "invalid_parameter",
        "message": "The 'success_url' must be a valid HTTPS URL.",
        "field": "success_url"
      }
    }
    ```
  </Accordion>
</AccordionGroup>

### Resource Errors

<AccordionGroup>
  <Accordion title="resource_not_found" icon="magnifying-glass">
    **HTTP Status:** 404
    
    **Description:** The requested resource does not exist.
    
    **How to fix:**
    - Verify the resource ID is correct
    - Ensure you're using the right environment (test vs. production)
    - Check if the resource was deleted
    
    ```json
    {
      "error": {
        "code": "resource_not_found",
        "message": "Product with ID 'prod_invalid123' not found."
      }
    }
    ```
  </Accordion>

  <Accordion title="duplicate_request" icon="clone">
    **HTTP Status:** 409
    
    **Description:** A resource with the same unique identifier already exists.
    
    **How to fix:**
    - Use a unique `request_id` for idempotent requests
    - Check if the resource already exists before creating
    
    ```json
    {
      "error": {
        "code": "duplicate_request",
        "message": "A checkout with this request_id already exists.",
        "field": "request_id"
      }
    }
    ```
  </Accordion>
</AccordionGroup>

### Rate Limiting

<AccordionGroup>
  <Accordion title="rate_limit_exceeded" icon="gauge-high">
    **HTTP Status:** 429
    
    **Description:** You've exceeded the allowed number of requests.
    
    **How to fix:**
    - Implement exponential backoff
    - Check the `Retry-After` header for wait time
    - Consider caching responses
    - [Contact us](https://creem.io/contact) for higher limits
    
    ```json
    {
      "error": {
        "code": "rate_limit_exceeded",
        "message": "Rate limit exceeded. Please retry after 45 seconds.",
        "retry_after": 45
      }
    }
    ```
    
    See [Rate Limits](/api-reference/rate-limits) for best practices.
  </Accordion>
</AccordionGroup>

### Payment Errors

<AccordionGroup>
  <Accordion title="payment_failed" icon="credit-card">
    **HTTP Status:** 422
    
    **Description:** The payment could not be processed.
    
    **How to fix:**
    - This typically occurs on the customer side
    - Provide clear messaging to the customer to try a different payment method
    - Check webhook events for detailed failure reasons
    
    ```json
    {
      "error": {
        "code": "payment_failed",
        "message": "The payment was declined by the card issuer."
      }
    }
    ```
  </Accordion>

  <Accordion title="subscription_not_active" icon="repeat">
    **HTTP Status:** 422
    
    **Description:** The subscription is not in an active state.
    
    **How to fix:**
    - Check the subscription status via the API
    - Subscription may be paused, canceled, or past due
    - Direct the customer to update their payment method
    
    ```json
    {
      "error": {
        "code": "subscription_not_active",
        "message": "Cannot modify subscription 'sub_xxx' because it is canceled."
      }
    }
    ```
  </Accordion>
</AccordionGroup>

### Webhook Errors

<AccordionGroup>
  <Accordion title="webhook_signature_invalid" icon="signature">
    **HTTP Status:** 401
    
    **Description:** The webhook signature verification failed.
    
    **How to fix:**
    - Ensure you're using the correct webhook secret
    - Verify you're using the raw request body for signature verification
    - Check that your server isn't modifying the request body
    
    ```typescript
    // Correct: Use raw body
    app.post('/webhooks', express.raw({ type: 'application/json' }), (req, res) => {
      const signature = req.headers['creem-signature'];
      const isValid = creem.webhooks.verify(req.body, signature);
    });
    ```
    
    See [Webhooks](/code/webhooks) for implementation details.
  </Accordion>
</AccordionGroup>

### Server Errors

<AccordionGroup>
  <Accordion title="internal_error" icon="server">
    **HTTP Status:** 500
    
    **Description:** An unexpected error occurred on our servers.
    
    **How to fix:**
    - Retry the request with exponential backoff
    - If the issue persists, [contact support](https://creem.io/contact)
    - Check our [status page](https://status.creem.io) for ongoing incidents
    
    ```json
    {
      "error": {
        "code": "internal_error",
        "message": "An unexpected error occurred. Please try again later."
      }
    }
    ```
  </Accordion>
</AccordionGroup>

## Troubleshooting Tips

### 1. Check the Environment

Make sure you're using the correct base URL:

| Environment | Base URL |
|-------------|----------|
| Production | `https://api.creem.io` |
| Test Mode | `https://test-api.creem.io` |

<Warning>
  Test mode API keys only work with `test-api.creem.io`, and production keys only work with `api.creem.io`.
</Warning>

### 2. Validate Request Format

```bash
# Check your JSON is valid
echo '{"product_id": "prod_123"}' | jq .

# Common issues:
# - Trailing commas
# - Single quotes instead of double quotes
# - Missing quotes around keys
```

### 3. Use the SDK

Our SDKs provide typed errors and better error messages:

```typescript
import { createCreem, CreemError } from 'creem_io';

const creem = createCreem({ apiKey: process.env.CREEM_API_KEY! });

try {
  const checkout = await creem.checkouts.create({
    productId: 'prod_invalid',
    successUrl: 'https://example.com/success',
  });
} catch (error) {
  if (error instanceof CreemError) {
    console.error(`Error ${error.code}: ${error.message}`);
    // Handle specific error codes
    if (error.code === 'resource_not_found') {
      // Product doesn't exist
    }
  }
}
```

### 4. Enable Logging

When debugging, log the full request and response:

```typescript
const creem = createCreem({
  apiKey: process.env.CREEM_API_KEY!,
  debug: true, // Logs all requests and responses
});
```
